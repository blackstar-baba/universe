## 描述
如何考察一个数据库底座：
1. 功能
    - 基础功能：语法、表、视图、物化视图、外表、索引、数据模型、函数、事务 、ETL（流批）、权限、隔离
    - 管理工具：部署、监控、 迁移、节点修复、数据备份及恢复等。
2. 非功能
    - 操作系统支持
    - 芯片架构
    - 语法兼容性： sql99\sql2003
    - 高可用
    - 负载均衡
    - 容灾：同城、异地
    - 生态：工具、插件
    - 扩展性：自定义函数、自定义数据类型、自定义数据源等
    - 性能
    - 稳定性
    - 已有系统迁移成本、已有技术生态是否适配
    - 信创适配范围
    - 背书：有哪些大型公司在使用
    - 开源度：开源部分有多少？是否真正开源？开源协议是什么
    - 复杂性：部署，运维
3. 技术架构：
    - 查询优化器：CBO/RBO
    - 执行引擎： 普通/向量化
	- 存储引擎： 行存、列存、行列混存
    - 架构类型： 存算分离/一体化
